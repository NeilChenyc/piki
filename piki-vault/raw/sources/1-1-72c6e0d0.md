### 用户篇

微信上所有的设计，都是面向用户的。产品经理应该像上帝一样建立一个系统，并制定规则，让群体在系统中自我演化。

**人的重要特性：**
- **人是环境的反应器** — 用户的行为受环境影响，产品要营造"环境"来牵引用户
- **人是懒惰的** — 懒惰催生发明，产品应减少用户操作成本
- **时尚是驱动力** — 推动微信普及的不仅是功能，更是时尚元素——"不用微信你就落伍了"
- **人是没有耐心的** — 不能让用户一分钟爱上产品，以后就不会再用了
- **人是不爱学习的** — "马桶阅读"理论：给用户的内容应短于马桶上的阅读时间
- **群体是"乌合之众"** — 群体智商不如个体智商，互联网产品面对的是群体而非个体
- **重视"草根"用户群** — 80%用户是草根，不能以周围同事代替用户画像
- **从日常体验中发现本质** — 伟大的产品应满足人的情感需求
- **人性化就是以己推人** — 通过自己理解人最普遍的心理
- **需求是满足人们的贪嗔痴** — 贪、嗔、痴是佛教三毒，产品的终极目标是满足这些

### 需求篇

**关于需求的十个观点：**
1. 对于新点子，99%的情况下否定是对的
2. 不要用户说什么就做什么
3. 从同类产品里找需求不是好办法
4. 不要听从产品经理个人的需求
5. 需求只来自你对用户的了解（不来自调研、分析、讨论、竞争对手）
6. 从微博感受用户潮流
7. 每个时代有每个时代的产品
8. 优先满足自己的需求（以己推人）
9. 找到需求背后的心理诉求
10. 心理满足的驱动力远胜工具甚至省钱

**经典案例：**
- "摇一摇"满足对随机性的好奇心，具备群体效应
- "附近的人"满足人天生对周围环境的好奇心
- 漂流瓶解决倾诉和发泄的欲望
- 朋友圈基于"人需要存在感"的认知
- 二维码是连接现实与手机的桥梁

### 设计篇

- 产品是进化出来的，而非规划出来的
- 有DNA的产品才会进化
- 把用户分为高中低端是不道德的
- 先做产品结构，之后才是功能细节
- 功能模块之间是有机联系的关系
- 设计就是分类
- 抽象方能化繁为简
- 面向场景来做设计，而非功能列表
- 让功能存在于无形之中
- 区别手机和 PC 的设计
- 不过度设计
- 宁愿损失功能也不损失体验
- 产品还是运营？做一劳永逸的事情
- 让用户推动用户
- 极简方能不被超越
- 操作的响应速度永远是第一体验
- 如果一个特性不能让人兴奋，不如不做
- 保留变化
- 如果没有自然增长，就不必推广
- 保持粗放，保持笨拙
- 在每个细节中体现认知
- 给用户选择权（插件化）
- 尊重用户
- 把用户体验做到极致就是创新
- 价值观决定产品的特性
- 避免战略行为替代真实需求
- 从人出发，改变商业模式
- 没有设计，只有解决问题

### 气质篇

- 改变旧世界的意愿
- 有自己的理念和愿景
- 微信是"一个生活方式"（不是"一种"）
- 让技术产品传达人文意识
- 产品中的文字反映气质
- 未来感的设计（未来、科技、抽象、简约）
- 你没有竞争对手
- 是作品，而不是产品
- 工匠，而不是设计师
- 做有灵魂的产品

### UI 篇

- UI 最重要的是条理清晰
- UI 围绕特性，而非特性围绕 UI
- 每个界面只有一个主题
- 对最常用的操作精益求精
- 需要文字解释的功能不是好体验
- 隐藏数字
- 探索手机上的新交互
- 于细微处用心
- 锻炼自己的审美能力
