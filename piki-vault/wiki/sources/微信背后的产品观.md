# 微信背后的产品观

## 这个来源说了什么

本书是 **张小龙** 在腾讯内部长达 8 小时的讲座整理稿，系统阐述了微信的产品哲学与设计方法论。全书分为用户篇、需求篇、设计篇、气质篇、UI 篇和提问环节六个部分，从对人性的理解出发，贯穿到具体的产品决策。

### 核心观点

1. **用户是人，不是数据** — 产品经理应像上帝一样建立系统、制定规则，让群体在系统中自我演化
2. **需求源于对用户的深度理解** — 需求不来自调研、分析、讨论或竞争对手，只来自对用户的了解
3. **群体效应是互联网产品的核心** — 为群体效应而做（如"摇一摇""附近的人""漂流瓶"），不可预知性才是产品的生命力
4. **极简主义** — "摇一摇"极简化到难以被超越；微信坚持只有 4 个 Tab 两年不变
5. **气质决定产品高度** — 微信不是一个工具，而是"一个生活方式"；作品而不是产品，工匠而不是设计师
6. **价值观即产品** — 不做"已读"、不做"iPhone 在线"、不做 PC 版，这些决策来自价值观而非功能分析

### 著名金句

- "我所说的，都是错的" — 全书收尾，提醒不要视为教条
- "产品是技术和艺术的结合" — 引用乔布斯
- "让你不计一切代价地展示聪明，还是选择善良？" — 引用贝索斯
- "少发微信" — 新版欢迎页，反营销式关怀
- "不听摇滚的程序员不是好的产品经理"

### 关键案例

| 案例 | 产品启示 |
|------|----------|
| "摇一摇" | 满足对随机性的好奇心，具备群体效应 |
| "附近的人" | 满足人对周围环境的好奇心，非为交友 |
| 漂流瓶 | 解决倾诉和发泄的欲望，默认发语音 |
| 朋友圈 | 基于"人需要存在感"，私密空间设计 |
| 不做"已读" | 保护用户隐私，减轻社交压力 |
| 二维码 | 连接现实与手机的桥梁，面向未来设计 |
| 语音提醒 | 以己推人，因自身需求而做 |

### 方法论框架

**需求来源优先级**：
1. 对群体效应的好奇（最高优先级）
2. 自身的需要和周边朋友的反馈
3. 对当下生活潮流的理解
4. 对信息流的理解
5. 对未来的理解
