# 微信背后的产品观

## 这个来源说了什么

本书是 **张小龙（Allen Zhang）** 在腾讯内部长达 8 小时的讲座整理稿，系统阐述了微信的产品哲学与设计方法论。全书分为 **用户篇、需求篇、设计篇、气质篇、UI 篇、提问环节** 六个部分，从对人性的理解出发，贯穿到每一个具体的产品决策。

### 书目信息

- **作者**：张小龙
- **主编**：陈妍、张军
- **出版社**：电子工业出版社，2021 年 1 月第 1 版
- **ISBN**：978-7-121-40252-4
- **字数**：194.4 千字

## 用户篇：人是环境的产物

张小龙开篇立论：**产品经理应像上帝一样建立一个系统，并制定规则，让群体在系统中自我演化。** 核心前提是——用户是人，不是数据。

### 人的七个重要特性

1. **人是环境的反应器** — 用户行为受所处环境牵引，产品要营造"环境"而非直接引导。案例：在计算机前写微博 vs 在路上写微博，内容截然不同。
2. **人是懒惰的** — 懒惰催生发明。案例：语音查找联系人——双手不便时用语音说名字即找到人。
3. **时尚是驱动力** — "今天如果不用微信，你就落伍了"，这种恐惧感比功能更强大。产品不能太"工具化"。
4. **人是没有耐心的** — 如果不能让用户一分钟爱上产品，以后可能就不会再来了。不要尝试教育用户。
5. **人是不爱学习的** — "马桶阅读"理论：给用户的内容应短于马桶上的阅读时间。
6. **群体是"乌合之众"** — 引用勒庞《乌合之众》、凯文·凯利《失控》。互联网产品面对的是群体而非个体，群体行为无法通过个体调研推导。
7. **重视"草根"用户群** — 80% 的用户是"草根"，不能以周围同事代替用户画像。

### 关键洞察

- **人性化就是以己推人** — 以上理论不是来自心理学学习，更多是"通过自己理解人最普遍的心理"。
- **需求是满足人们的贪嗔痴** — 贪（贪婪）、嗔（羡慕嫉妒恨）、痴（沉迷）。产品的终极目标是满足这些，不在意道德评价。微信新版建议"少发微信"正是基于此的关怀。
- **伟大的产品应满足人的情感需求**，而不仅是提供功利上的好处。案例：三星 Note 成功是因为时尚 + 输入舒适；小音响的触控开关是情感交互。
- **让不上微博的人来用你的产品** — 上微博的人代表有探索意愿的群体，让更懒惰的群体也来用，才是真正的用户主体。

## 需求篇：需求不来自调研

### 关于需求的十个观点

1. **对新点子，99% 否定是对的** — 臆想需求会引发风险，否定是安全的默认选择。
2. **不要用户说什么就做什么** — 用户反馈帮你了解想法，但需求是零散的，解决方案需要归纳抽象。
3. **从同类产品里找需求不是好办法** — 你看不到别人的思考过程。案例：微信坚持不做涂鸦功能。
4. **不要听从产品经理个人的需求** — 产品经理经过理性训练反而不能代表大众的朴素心理。
5. **需求只来自你对用户的了解** — 不来自调研、不来自分析、不来自讨论、不来自竞争对手。调研可以改进细节，但绝不会告诉你"做一个摇一摇"。
6. **从微博感受用户潮流** — 每天花一小时看普通用户在微博上谈论如何用你的产品，感受的是"潮流"而非具体数据。
7. **每个时代有每个时代的产品** — QQ 是 PC 互联网时代的产品，微信是移动互联网时代的产品。感知不到时代变化就停留在过去。
8. **优先满足自己的需求（以己推人）** — 理解不了那么多人，只能从自身理解来想象。案例：不做"已读"是因为自己不想被知道已读。
9. **找到需求背后的心理诉求** — 不是在做一个功能，而是要满足用户的诉求。案例：漂流瓶是为了倾诉和发泄，不是交友。
10. **心理满足的驱动力远胜工具甚至省钱** — "可以看到附近有哪些帅哥美女"比"每月省 10 块钱"更有吸引力。

### 核心方法论

| 原则 | 说明 |
|------|------|
| **"爽"胜过功能** | 产品能否让用户说"爽"或"好玩"是口碑传播的关键。省钱不会让人主动传播，好玩会。 |
| **为群体效应而做** | 建立规则让群体自我演化，不可预知性是生命力。典型案例："附近的人""摇一摇""漂流瓶"。 |
| **为想法而做** | 不是因为竞品有所以做，而是自己想用。朋友圈的起点：人都需要存在感。 |
| **只抓主场景，不做全功能** | 朋友圈默认只能发照片，纯文字入口隐藏在深处——因为普通人发 140 字很困难。 |
| **最后一刻才决定新版本特性** | 微信每个版本之间没有任何计划。一个月前的想法和今天已有区别，最后一刻的决定才能反映最新思路。 |
| **世界是新的** | "PC 上的入口在搜索框，手机上的入口在二维码。"忘记过去的数据甚至经验，洞察未来趋势。 |

### 需求来源优先级

1. 对群体效应的好奇（最高优先级）：做之前不知道会发生什么
2. 自身的需要和周边朋友的反馈：案例 — 语音提醒、蓝牙耳机、Web 微信（定位为"连接键盘"而非 PC 版）
3. 对当下生活潮流的理解：从微博上感知
4. 对信息流的理解：微信应该成为 Push 中心，所有信息自动推送给用户
5. 对未来的理解：比当前的需要更重要 — 二维码系统的建立即是面向未来

### 经典案例集

| 案例 | 需求本质 | 产品启示 |
|------|----------|----------|
| **摇一摇** | 对随机性的好奇心 + 同时刻做同一动作的群体效应 | 满足人性驱动，又有群体效应 |
| **附近的人** | 人天生对周围环境的好奇心 | 不是为了交友，而是"看到"是第一需求 |
| **漂流瓶** | 倾诉和发泄的欲望 | 默认发语音——语音比文字更直接迅速 |
| **朋友圈** | 人需要存在感和朋友间的私密空间 | 基于"人总是需要三五个好友"，私密设计 |
| **石头剪刀布** | 来自微博的一条帖子"已经没有什么可以阻止人类用微信语音玩石头剪刀布了" | 需求不来自用户直接告诉你要什么 |
| **不做"已读"** | 保护隐私、减轻社交压力 | 价值观决定产品特性 |
| **不做 PC 版** | 破坏"永远在线"的信赖感 | Web 微信定位为"连接键盘"而非 PC 版 |
| **二维码** | 连接现实与手机的桥梁 | 面向未来的系统建设 |

## 设计篇：极简方能不被超越

### 核心理念

- **产品是进化出来的，而非规划出来的** — 有 DNA（价值观和认知）的产品才会进化
- **把用户分高中低端是不道德的** — 好的产品应该所有人都喜欢用。"摇一摇"不低俗，出发点纯洁即可
- **先做产品结构，之后才是功能细节** — 微信功能非常多，但看起来简单，因为骨骼梳理清楚了
- **功能模块之间是有机联系的关系** — 朋友圈 + 个人相册 = 不需要多头像功能

### 设计方法论

| 方法 | 解释 | 案例 |
|------|------|------|
| **设计就是分类** | 大部分工作做不好是分类分不好 | 微信 4 个 Tab，两年不变（与 Tony 的君子协议） |
| **抽象方能化繁为简** | 100 个需求抽象成 10 个，从 10 个再派生 100 个 | 微信公众平台用一种账号体系支持所有内容类型 |
| **面向场景来做设计** | 功能脱离场景没有意义 | 语音自动播放：只有开车/独处时合适，开会时尴尬→不做 |
| **让功能存在于无形之中** | 不把新功能摆出来，藏着才是最好的 | 商家会员：扫二维码才呈现，在微信里看不见 |
| **不过度设计** | 不要表现自己的聪明才智 | "附近的人"最初连性别都没有，不做过多的修饰 |
| **宁愿损失功能也不损失体验** | 权衡时体验优先 | 微信不设"编辑"按钮——怕误触返回操作 |
| **给用户选择权（插件化）** | 核心功能以外皆可卸载 | 朋友圈、语音视频聊天都是可卸载的插件 |

### 重要原则

- **操作的响应速度永远是第一体验** — 邮箱成功的两个关键词："简单"和"快捷"
- **如果一个特性不能让人兴奋，不如不做** — 好奇心是产品经理的驱动力
- **保留变化** — 不在 1.0 做 2.0 的事。朋友圈第一版连评论回复都没有，因为"可以放到以后再做的"
- **如果没有自然增长，就不必推广** — KPI 只是好产品的副产品。微信 2011 年 5 月才开始推广，在此之前零广告
- **保持粗放，保持笨拙** — 没有好的解决方案时先不做。案例：发错群消息的问题至今没有好方案
- **如果在每个细节中体现认知** — 不做"来自 iPhone 客户端"这种无含义信息
- **尊重用户** — 隐私保护、数据保护、不用"系统管理员"发通知信
- **把用户体验做到极致就是创新** — 微信"放到耳边自动切听筒"是通过动作识别而非感光做到的，灵感来自 Siri
- **你的价值观决定产品特性** — 要不要滤镜？要不要 iPhone 在线？取决于态度而非分析
- **改变用户习惯** — 视频通话需要对方"同意"才能拨打（取代传统电话的强行中断）
- **抛弃不人性的创新** — 用耳机接听键当"按住说话"按钮？会使用户分裂——不做
- **避免战略行为替代真实需求** — 导入 Qzone 照片会破坏朋友圈氛围。50 分产品合起来低于 50 分
- **从人出发，改变商业模式** — 品牌通过朋友传播，而非传统广告。微信"关注"品牌功能由此而来
- **没有设计，只有解决问题** — 设计不是高尚的职业，只是把问题解决好

### 关于创新与抄袭

- 所有创新都来自知识的积累，不存在凭空创造。"摇一摇"用了加速度传感器，不能说因为别人也能用就不能做
- 跟随用户而非跟随业界产品。当你达到一亿用户时，用产品说话比辩解更有威力

## 气质篇：做有灵魂的产品

### 核心理念

- **改变旧世界的意愿** — 张小龙早在 2010 年就期待一台无电话功能的手机（用 Kik 短信、Google Voice 通话、FaceTime 视频），如今已经实现
- **有自己的理念和愿景** — "微信是一个生活方式"（注意是"一个"不是"一种"——"个"更艺文）
- **是作品，而不是产品** — 作品可以打败功利产品。工匠，而不是设计师
- **你没有竞争对手** — 不分析竞争对手，关注自己想要什么、关注未来

### 人文表达

- **让技术产品传达人文意识** — QQ 邮箱登录页用过马丁·路德·金的话、海子的诗、中国第一封电子邮件纪念
- **微信启动画面** — 一个小人站在月球上看地球。张小龙说"能让用户有不同的感觉就对了"。孤独是永恒主题
- **"少发微信"** — 除烟以外，还有哪个产品让自己用户少用？这是一种反营销式的关怀
- **致敬 MJ** — 在微信某版本回应外界批评，用 MJ 的音乐表达"如果你说我是错的，最好证明你是对的"
- **欢迎页不罗列功能** — "新功能的罗列是极端不好的"，欢迎页是表达感受的地方
- **文字反映气质** — 清晰易懂（有逻辑）、不讨好用户、不抬高自己、把用户当朋友。不用"吧""哦""您"、不用"已保存成功"（难道还有"已保存失败"吗？）

### 未来感的设计

- 微信视觉关键词：**未来、科技、抽象、简约**
- 视频通话背景原本是蓝天白云——太俗气。应该抽象化而非模拟现实

### 做有灵魂的产品

> 产品应该像人一样，有很多面，有机联系：
> - **骨骼**：产品结构
> - **肌肉**：功能
> - **气质**：价值观
> - **反应**：理性、逻辑清晰、交互合理
> - **谈吐**：文案

精神分裂的产品：这边来自这种想法、那边来自那种想法，互相妥协。保持对产品的坚持甚至独断才能使产品不分裂。**集体决策会导致平庸。**

## UI 篇：条理清晰最重要

### 核心原则

- **UI 最重要的是条理清晰**，不是绚丽的视觉效果。微信用系统控件，保持一致的式样
- **禁止使用竞品** — 广研（微信事业群前身）禁止 UI 设计师使用竞争对手产品，避免受其影响
- **UI 围绕特性，而非特性围绕 UI** — 不要为了一个 UI 点子去编一个功能
- **每个界面只有一个主题** — 学习摄影：一张照片里只有一个主题
- **每个对话框都有一个明显的默认按钮** — 右上角完成按钮，蓝色，显眼
- **对最常用的操作精益求精** — 聊天列表下拉到顶端的转动动画确保平滑
- **需要文字解释的功能不是好体验** — 最好不用 Tips。每次用 Tips 都是因为做得不好
- **隐藏数字** — 不精确显示"78.56%"，不显示微信有多少个联系人
- **探索手机上的新交互** — 手机有更多传感器，值得探索
- **于细微处用心** — 会话框下拉空白处填上微信 LOGO
- **锻炼自己的审美能力** — 研究 iPhone 的设计历程，观察身边的好设计（音响、iMac 等）

## 提问环节精选

### Q：微信有硬伤吗？
张小龙认为任何产品都不会有致命缺陷，因为一旦出现就会被弥补。

### Q：人人都是产品经理？
"人人可以提问题、指手画脚，但能把问题整理出来、找到本质的东西是很难的。这才是产品经理应该走的路。"

### Q：从一个普通工程师做成伟大产品，怎么走过来？
**一万小时定律**：一个人要成为领域专家，需要一万小时的专业训练。"经历上千个实战的锻炼，越多越好。"约翰·列侬、莫扎特都不是天赋神童，而是长时间专业训练的结果。

## 结尾名言

> **"我所说的，都是错的。"**

张小龙提醒：不要把这些话当教条。"任何东西一旦说出来，变成了一个教条，它就已经错了。一定是你自己理解到的才是你自己的。"
