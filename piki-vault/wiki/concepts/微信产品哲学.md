# 微信产品哲学

微信产品哲学是 **张小龙** 在打造微信及其他产品（Foxmail、QQ 邮箱）过程中形成的一套系统化产品方法论。其核心是 **从人性出发**，将产品视为有灵魂的作品而非工具。

## 核心理念

### 1. 人性驱动设计
- 接受并利用人的弱点：懒惰、贪嗔痴、缺乏耐心、不爱学习
- 用户不是理性"经济人"，而是被环境、情感、群体心理驱动的"人"
- 通过理解自己来理解用户（"以己推人"）

### 2. 群体效应优先
- 互联网产品面对的是群体而非个体
- 产品应建立规则和环境，让群体在其中自我演化
- 不可预知性和不可测性是产品的生命力所在
- 案例："摇一摇""附近的人""漂流瓶"——做之前不知道会发生什么

### 3. 极简主义
- "极简方能不被超越"
- 分类和抽象是设计最核心的方法
- "宁愿损失功能也不损失体验"
- 做"类型"而非做"实例"

### 4. 气质与价值观
- 产品要有灵魂，像人一样有多个有机联系的"面"
- 价值观决定产品特性（不做"已读"、不做"iPhone 在线"）
- "是作品，而不是产品"
- "工匠，而不是设计师"

### 5. 面向未来
- "世界是新的"
- 二维码是手机时代的入口
- 对未来场景的理解比当前需求更重要

## 方法论工具

| 工具 | 描述 |
|------|------|
| 场景设计 | 面向场景而非功能列表做设计 |
| 抽象化 | 100个需求抽象为10个再派生 |
| 插件化 | 核心功能以外皆可卸载 |
| 自然增长 | 无自然增长则不推广 |
| 保留变化 | 不在1.0做2.0的事 |

## 与 KPI 文化的关系

张小龙主张 **跳出 KPI 做产品**，认为 KPI 只是好产品的副产品。如果工作围绕 KPI 指标，会变得无趣且失去发现。"好的产品应该是自然增长的，而不应该为 KPI 而改变产品。"
