# Piki 开发过程总览

这份文档用于整理 Piki 在开发过程中形成的中间认识，方便后续继续实现时快速回到上下文。

## 文档分类

### 1. 产品文档

这部分回答“我们要做什么”。

- `../product/PRODUCT_OVERVIEW.zh.md`
  - 总体产品定位、目标人群、核心痛点和 MVP 切入点
- `../product/MVP_FEATURES.zh.md`
  - MVP 功能全景、分层能力地图、各模块能力范围
- `../product/ROADMAP.md`
  - 开发阶段拆分、交付物、验收标准和暂不做范围
- `../product/PRODUCT_NOTES.md`
  - 产品笔记、命令层设计、对外部参考的吸收

### 2. 开发过程文档

这部分回答“开发过程中发现了什么问题，以及怎么解决”。

- `issues/长文-ingest-问题与解法.md`
  - 长文 ingest 为什么会漏读
  - 为什么不能依赖一次上下文直接总结
  - 应该怎样把 ingest 改造成可验证、可追踪的编译流水线

## 推荐使用顺序

1. 先看 `../product/PRODUCT_OVERVIEW.zh.md`
2. 再看 `../product/MVP_FEATURES.zh.md`
3. 接着看 `../product/ROADMAP.md`
4. 需要补实现细节时看 `../product/PRODUCT_NOTES.md`
5. 遇到具体实现问题时，再进入 `issues/` 目录查对应文档

## 后续建议

- 每出现一个值得保留的实现问题，都在 `issues/` 下新增一篇单独文档
- 每篇 issue 文档尽量统一结构：背景、现象、原因、方案、对产品的启发
- 当某个问题已经进入正式方案，可以在产品文档中补链接，形成“问题 -> 方案 -> 路线图”闭环
